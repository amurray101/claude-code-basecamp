[TRANSITION] "Let me use an analogy you already understand."

"Before USB, every device had its own proprietary connector -- every phone had a different charger cable. USB created a universal standard that any device could use. MCP is the USB of AI. It is an open protocol that lets any AI model connect to any tool through a single standard interface."

[KEY MOMENT] "The Model Context Protocol -- MCP -- was created by Anthropic and released as an open standard. That is a strategic choice: Anthropic created it and open-sourced it, which is a strategic moat disguised as generosity -- similar to how Google created Kubernetes and open-sourced it."

MCP Architecture

Three layers, fully plug-and-play

Host
Claude Desktop · Claude.ai · Claude Code
Client
Manages connections to MCP servers
Servers
Gmail · Calendar · Drive · Slack · Notion · GitHub · ...

Same server works with any host. Same host works with any server. True plug-and-play.

[TRANSITION] "Here is the architecture in three layers."

Walk through bottom-up: "At the bottom, MCP Servers wrap external tools -- Gmail, Calendar, etc. -- and expose their capabilities through a standardized interface. In the middle, the Client manages connections to these servers. At the top, the Host is the AI application -- Claude Desktop, Claude.ai, Claude Code."

"The key insight: the same MCP server works with any host, and the same host works with any server. If someone builds a great Salesforce MCP server, it instantly works with Claude Desktop, Claude.ai, AND Claude Code."
