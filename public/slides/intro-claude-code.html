Part 3 · Exercise 1

Create the Page

💬 Type this into Claude Code
“Build me a personal landing page. Include my name, a short intro about what I’m interested in, and a way for people to get in touch. Make it look clean and modern.”

Make it yours — use your real name, or make something up. Then accept the changes and open the file in your browser.

What to watch for: Claude will pause to think, then create files. This is the Read → Plan → Act loop from Part 1, happening in front of you.

[TIMING] Give people 5-7 minutes. Walk around. Common issues: people not accepting the changes, not knowing how to open the HTML file in a browser (right-click → Open With), the extension not being signed in.

[IF IT GOES WRONG] If Claude doesn't create any files, they may not have accepted the changes — look for the green checkmarks in the diff view. If the page looks broken, have them tell Claude: "The page isn't displaying correctly — can you check the HTML?" If the extension isn't responding, try reloading VS Code.

[NARRATE] "Notice how Claude read the empty directory, planned what to create, and then built the whole page — HTML, CSS, and content. One prompt. That's exercise 1."

Part 3 · Exercise 2
~7 min

Add Interactivity

💬 Type this into Claude Code
“Add a dark mode toggle to the page. It should switch between light and dark themes, and remember my choice when I come back.”

Accept the changes and refresh your browser. Try clicking the toggle. Close and reopen the page — does it remember?

What to watch for: Claude reads the existing page before making changes — it doesn’t start from scratch. If something doesn’t work, that’s Claude’s mistake, not yours. Just tell it what’s wrong.

[TIMING] 5-7 minutes. This exercise shows two things: (1) Claude reads existing code before editing, and (2) follow-up requests build on previous context.

[IF IT GOES WRONG] Perfect teaching moment! If the toggle doesn't work, have them tell Claude: "The dark mode toggle doesn't seem to be working — can you check it?" This demonstrates the correction loop: it's OK to tell Claude something is wrong.

Wrap-up

Three Things to Watch Out For

Now that you’ve tried it, these will make more sense

⏳ Long conversations get fuzzy

If you keep adding to the same session for a long time, quality drops. Start fresh for new tasks. Our three exercises worked well because each was focused.

🎯 Vague requests get vague results

Notice how our prompts said exactly what we wanted? “Add a dark mode toggle that remembers my choice” works. “Make it better” doesn’t.

👋 It’s not always right — and that’s OK

If something didn’t work, that’s Claude’s mistake, not yours. Just tell it what’s wrong and let it try again. The more you correct it, the better it gets. That’s a skill, not a failure.

[GROUNDED] "These aren't abstract warnings anymore — you've seen the tool. You know what 'be specific' means because you just did it. You know what 'start fresh' means because you felt the session context building."

Wrap-up

Good to Know

👑 You’re still in charge. Claude proposes changes. You review and approve them.

🌱 It gets better with context. The CLAUDE.md file tells Claude about your real project — your conventions, your tools, your patterns.

🤝 It’s a tool, not a replacement. It handles the tedious parts so you can focus on the interesting ones.

[TONE] This slide builds trust. They've now seen it work — they know it's real. But they may still have concerns about control and relevance. Address both.
